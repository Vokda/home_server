Vim

A text editor. And in order to contribute to the eternal flame war: The very best editor. Fight me, Emacs users!

Jokes aside, I'm sure we can all agree they are both great text editors in their own way. You can find Vim here and Emacs here.

As mentioned earlier, Vim is a text editor. That is a progam which one edits texts with. Vim stands from Vi Improved. Vi in turn is an even older text editor.

Why use an ancient text editor you ask? Because it runs everywhere. It's a small editor that does not require much of your computer. It is very useful if you only have (or only want) to do things in a terminal.

Vim is not the easiest text editor to start out with, but if you take your time to learn it properly you'll never want to go back. It's probably the same with Emacs. I don't since I never bothered to learn it.

Personally I Neovim (or nvim for short) if it's available. It's a modernized verison of vim if you will.

One point of critique against Vim: It's regex is all kinds of fucked up. It should follow the Perl standard.
